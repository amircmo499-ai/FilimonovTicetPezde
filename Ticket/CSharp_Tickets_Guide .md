
#  Основы программирования 


## 1.1 Переменные и типы

 ### object
   Все типы данных наследуются от него он считается прародителем всех типов потому что в нем может завернут  любой тип. 

  * **boxing (упаковка)**
   Когда мы кладем какойто тип например число в переменную тип object, происходит  упаковка (boxing) и  из стека (Stack) переносится в кучу (Heap), упаковка (boxing) сьедает  часть ресурсов 

  * **Unboxing (распаковка)**
   Это какраз противоположность boxing (упаковка) это когда мы переводим тип object  в в какойто другой значимый тип  например чисор  и из кучи (Heap) переносится в стек (Stack) это сьедает  часть ресурсов также GB колектор использует часть ресурсов для очистки в куче (Heap)

 ### Преобразование типа
   Это когда при образование  создается новый обьект  
   пример была  строка в которой было число  45 на основе этой строки в памяти создался новый обьект типа  число в которой уже было значение 42 не в виде  строки а ввиде числа
   
  

  #### TryParse/Parse
  Это  методы  которые  предназначены для конкретного преобразования строки  в какойто тип если строка будет  равна null то  кинет исключение

  * **Parse** 
  это метод который принимает   строку и возращает  нужный тип (не безопастный если  например  будет строка "das" и мы захотим ее в число то  кинет ошибку)

  **синтаксис Parse:**

    int result = int.Parse("12345");
  
  * **TryParse** 
  это метод который принимает   строку и записывает в переменую  значение  переменая создается в скобках с помощью  ключевого слова [out](#ключевые-слова) (он  безопастный если будет строка "das" и мы захотим ее в число то  вернет  false и запишет в переменую дефолдное значение для типа)

  **синтаксис TryParse:**

    int.TryParse("12345",out int result);

  #### Convert
  Это класс который в отличии от TryParse/Parse может  преабразовывать не только строку но и другие типы например из bool в int  также если не боится null значений  также если не удалось переобразовать например пустую строку в число кинет исключение

  **синтаксис:**

    int result = Convert.ToInt32("123");

  
---

 ### Приведение типа 
   Это когда мы  переводим из  одного типа в другой  например  из  [object](#object)  в int  или  из double  в int 

  #### безопастное приведение 
   Это когда мы приводим тип так чтоб програма не вылетала с ошибкой для этого используются операторы   as  и is

  * **оператор as**
   это оператор  используется для безопастного приведения  логика такая если удалось привести  тип  то присваивает значение если нет то  присваивает Null

    **синтаксис:**

        string str = obj as string;

    где
    string - (в начале) тип переменой к которому хотим привести 
    str - название переменой 
    obj -  переменная в которой  есть возможное значение для нашей переменой  
    string - (в конце) тип к которому приводим
    

  * **оператор is**
    это оператор  используется для безопастного приведения  логика такая если удалось привести возращает true  если нет то возращает  false (работает в конструкции  [if else](#конструкция-if-else)

    **новый синтаксис:**

        if(obj is string str)
        {

        }
    
    
    
    **старый синтаксис:**

        if(obj is string)
        {
        str = (string)obj;  
        }

    где
    string -  тип переменой к которому хотим привести 
    str - название переменой 
    obj -  переменная в которой  есть возможное значение для нашей переменой  
    
  #### не безопастное приведение 
  Это когда мы приводим тип   и может кинуть исключение  например  неявное приведение  и явное приведение (стиль языка С) 

  * **неявное приведение** 
  это когда мы  моем не указывать к какому типу мы приводим например  из меньшего числа в большее из byte  в int  компилятор сам приводит тип 

  **синтаксис:**
  
    byte age = 17;
    int num = age

  * **явное приведение** 
  это когда мы прямо говорим к какому типу мы приводим  указывая тип в переди переменой  в скобках эта тема держиться  еще с языка С и она как раз может выдать исключение

  **синтаксис:**
  
    double number = 23.33;
    int num = (int)number //в переменой num будет 23

 ### string (строки)
   строка - это  масив символов  (char)  поэтому мы можем  к строке обращаться как к [массиву](#14-массивы)  по индексу но строка является immutable тоесть не изменяемой а если еще понятней мы не можем как в масиве по интексу изменить символ мы можем только посмотреть 

   * [Метод](15-методы) Substring 
   это метод который принимает 2 параметра  типа int и возращает новую строку из части нашей строка указаную в диапазое 

   **синтаксис:**

     string  name = "Egor"; //например мы хотим вывести 3 символа  без первого символа
    Console.WriteLine(name.Substring(1,3)) // выведет gor

 
## 1.2 Конструкторы ветвления 

---

### Конструкция if-else 
  это Базовое ветвление с помощью которой  можно гибко управлять логикой  
    
   **блок if:** в этот блок мы попадаем если  условие  было верно тоесть вернуло true (может существовать без else  и else if )
    
   **блок else if:** это дополнительное условие  работает также как и if  выполнится если условия выше не выполнились  колличество не ограниченое (без  if  не может существовать) 
    
   **блок else:**  этот блок выполнится если ни одно условие выше не выполнилось 
    
   **синтаксис if else  без else if:**

    if(условие)
    {
        //если  условие вернет true  то попадем в этот блок
    }
    else
    {
        //если  условие вернет false  то  попадем в этот блок 
    }


   **синтаксис if else вместе с else if:**

    if(условие)
    {
        //если  условие вернет true  то попадем в этот блок
    }
    else if(условие)
    {
      // если  есловие в if  вернуло false  и условие в  else if  вернуло true  попадем в этот блок
    }
    else
    {
        //если  условие вернет false  то  попадем в этот блок 
    }


   **синтаксис if без else и else if:**

    if(условие)
    {
        //если  условие вернет true  то попадем в этот блок
    }


   **синтаксис if else вместе с else if (краткая запись она работает только если у нас одна строчка кода должна быть в блоке):**

    if(условие)
     //если  условие вернет true  то попадем в этот блок
    else if(условие)
      // если  есловие в if  вернуло false  и условие в  else if  вернуло true  попадем в этот блок
    else
      //если  условие вернет false  то  попадем в этот блок 
   

   ---

 ### Тернарный оператор
   это вид компактной записи [if-else](#конструкция-if-else). всегда возращает какоето значения и мы должны его записать кудато
   
   там может быть только if и else   там не  может быть дополнительного условия  else if  или if  без else

   **синтаксис:**

    условие ? значениие_значение_если_true : значениие_значение_если_false;

   **пример:**

    string result = 5 < 18 : "да" : "нет";
   
---

 ### Switch-case
   это тоже ветвление с помощью которого можно сравнить на равенство  
   
   **в switch** записывается значение которое будем сверять 
   
   **в блок case**   попадает если условие было равно  также  в  case  блок закрывается (блоков case может быть сколько душе угодно) ключевым словом break 
   
   **в блок default**  попадает если  мы не попали ни в один блок  case  также default блок закрывается  ключевым словом break
   
   **примечание** блок case можно обьединять  вместе и получить вместо  проверки равно проверку на диапазон

   **синтаксис (без диапазона и без блока default):**

    switch(значение)
    {
     case значение с которым сверяем: 
        //если  совпало то  попадаем в блок
        break:
     case значение с которым сверяем: 
        //если  совпало то  попадаем в блок
        break:
    }


   **синтаксис (c блоком default без диапазона ) :**

    switch(значение)
    {
     case значение с которым сверяем: 
        //если  совпало то  попадаем в блок
        break:
     default:
       // если мы не попали не в один блок case то попадем в этот
        break:
    }


   **синтаксис (c блоком default и диапазона ):**

    switch(значение)
    {
     case значение с которым сверяем: 
     case значение с которым сверяем: 
     case значение с которым сверяем: 
        //если  совпалj то  попадаем в блок
        break:
    case значение с которым сверяем: 
     case значение с которым сверяем: 
     case значение с которым сверяем: 
        //если  совпало то  попадаем в блок
        break:
     default:
       // если мы не попали не в один блок case то попадем в этот
        break:
    }

## 1.3 Циклы: 

   цикл - Это конструкция  которая может выполнять повторно один и тотже блок кода  пока условие   true 

 ---
 
 ### Виды цикла
  Всего 4 вида циклов  в  с#
  
  * **while**
   базовый вид цикла  которому дают условие и пока оно верно  блок кода будет выполняться повторно

   **синтаксис:**
   
    while(условие)
    {
     //блок который будет выполнятся пока условие true
    }


  * **do while**
  работает практический также как и  while  только  он выполнится 1 раз даже если условие  не true тоесть мы сначало выполняем блок и потом проверяем на условие 

  **синтаксис:**
  
    do
    {

    }
    while(условие);


  * **for**
  это цикл со счетом  тоесть мы можем запихнуть туда не только условие а  и переменую с которой мы сможем работать только внутри блока в цикле   и также  что с этой переменой будет происходить работает так прошла итерация   и счетчик поменялся например уменьшился и так с каждой итераций, обычно используют для работы с [масивами](#14-массивы)

  **синтаксис:**

    for(счетчик;условие;шаг)
    {

    }

  **Дополнение:** мы можем не писать  например  счетчик или шаг или даже условие  может быть выглядеть for(;условие;шаг) или for(счетчик;;шаг) или for(счетчик;условие;) или еще 
  for(;;)

  **пример:**
  
    for(int i = 0; i < 10; --i)
    {

    }

  * **foreach**
  это цикл работающий только с обьектами которые в нутри себя реализуют интерфейс [IEnumerable]()  такие как [масивы](#14-массивы) коллекции и тд
  как работает просто перебирает элементы

   **синтаксис:**

    foreach(тип название_переменой in  коллекция или масив)
    {

    }

   **пример:**

    int[] arr = {2,4,6,2};
    foreach(int i in arr)
    {

    }
    
---

  ### Вложенные циклы
  это когда  в цикле  приписан еще цикл  это можно сделать с любым циклом 

  **пример с for:**
  
    for(int i = 0; i < 10; --i)
    {
      for(int j = 0; i < 10; --j)
      {
        
      } 
    }


## 1.4 Массивы
 Массив — однотипные значения находящиеся  в Heap (куче). у которого название переменой указатель на первый элемент  в масиве 
 
---

 ### Виды масивов (Часто используемые)

 
 * **Одномерные массивы Линейная структура**

  одномерный масив распалагается в оперативной памяти последовательно тоесть каждый элемент масива идет  друг за другом 

  выглядит как линия  из чисел


  **синтаксис:**

    тип[] название_переменой = new тип[колличество элементов];

   **синтаксис:**

    тип[] название_переменой = {значение,значение,значение и тд};


 * **Двумерные массивы: Табличная структура (строки и столбцы)**

    двухмерный масив  как и одномерный только он хрантся по принципу что  первая последовательность элементов у казывает на следующую последовательность элементов

    выглядит как таблица

    **синтаксис:**

       тип[,] название_переменой = new тип[колличество строк,колличество столбцов];

    **синтаксис:**

       тип[,] название_переменой = 
       {
          {значение,значение,значение и тд}
          {значение,значение,значение и тд}
          {значение,значение,значение и тд}
           и тд
       };
   

  ### Полезные свойства/методы масива

 * [Свойство](#свойство#) Length: Возвращает общее количество элементов в массиве .
 * [Метод](15-методы) GetLength(i): Возвращает длину конкретного измерения ( обычно для многомерных масивов используется 0 — строки, 1 — столбцы).
 * Min(): [Метод](15-методы) из библиотеки [LINQ](#37-язык-запросов-linq), пробегает по всем ячейкам и ищет меньшую.


## 1.5 Методы 
  Метод - это  именованный блок кода который можно вызывать по имени .

  **синтаксис:**

    возращаемое_значение имя_метода(принимающие_параметры)
    {

    }
   


 ### Передача данных в методе
 * **Принимающий параметр**
   
  Данные, которые мы передаем в метод
   это  нужно чтобы метод мог работаь со значенимяи которые нам нужны
    метод может принмать любое количество параметров сколько вы пропишите 
    и любой тип также можно не указывать что метод будет принимать 

   **пример с  типом int:**
   
    возращаемое_значение  имя_метода(int num,int num1 и тд)
    {

    }


   **пример без принимающего параметра:**

    возращаемое_значение  имя_метода()
    {

    }
   

 * **Возвращаемое значение:** 
  Результат, который метод вернуть обратно через return тип результата может быть любой. Если результата нет  используем void в данном случае return можно не писать.

  **пример с типом int:**

    int  имя_метода(принимающие_параметры)
    {
      return результат_типа_int;
    }

  **пример с void:**

    void имя_метода(принимающие_параметры)
    {
      return результат_типа_int;
    }
  

  
  
  ### Доп функционал
 * **Перегрузка метода:** перегрузка это когда у метода одно и тоже название а тип или колличество параметров разное  и в зависимости что методу передадите та реализация будет 
   
   **пример:** 

       void Summ(int num1, int num2)
       {

       }
       void Summ(int num1, int num2, int num3)
       {

       }


 * **Лямбда-выражения:** Анонимные функции (метода без названия) . может использоватся для [делегата](#)  и  как сокращения для [метода](#15-методы) и еще много для чего.

  **пример лямды:**
   
    (принимающие параметры) => 
     {
       // логиика
     }

   **пример лямды (если лямда написана в одну строку то компилятор сам  додумается что возращать  так что return ненадо писать):**
   
     (принимающие параметры) => //логиика;


   **пример для работы с методами ( для сокращения если логика в одну строку):**


    возращаемое_значение  имя_метода(принимающие_параметры) => логика;

 * **Рекурсия:** Метод, вызывающий сам себя. Требует обязательного (условия выхода), иначе возникнет [StackOverflow]().

  **пример с типом int:**

    int Function(int i) 
    {
       if(i<=0) return 1;//обязательно  условие для выхода а то  переполним стек

      return 2-FUnction(i-1);//возрат и вызов данного метода
  }
 
 
 
  ### Ключевые слова 
   * ref 
   Передает переменную по ссылке. Метод может как читать её значение, так и изменять его. Все изменения сразу отражаются на исходной переменной.

   * out
   Переменная может быть не инициализирована до вызова, но метод обязан присвоить ей значение внутри себя до завершения работы.

   * in
   Передает переменную по ссылке, но запрещает её изменять внутри метода (делает её доступной только для чтения)


   
# Объектно-ориентированное программирование (ООП)


### 2.1 принципы  ООП
 1. **Инкапсуляция:** Скрытие внутренней логики и защита данных (с помощью private и свойств).
 2. **Наследование:** Создание новых классов на базе существующих класса или интерфейса.
 3. **Полиморфизм:** один и тотже обьект может использовать разное  поведение .
 4. **Абстракция:** Выделение только значимых характеристик объекта, отсекая лишние детали реализации.

## 2.2 Класс

 ### Класс
 это некий Шаблон/Чертеж будущего объекта.

**пример:**
 
    class  имя_класса
    {

    }


 ### Метод класса
 Обычный [метод](#15-методы) который привязан к  классу и может  работать  с  его членами в нутри  даже с private

 **пример:**
 
    class  имя_класса
    {
      public void Print() // метод который принадлежит класса
      {

      }
    }

 ### Поле
 переменная класса в нутри класса.

**пример:**
 
    class  имя_класса
    {
      public int num; //поле
    }
   
### Свойство
некий синтаксический сахар  обьединяет в себе логику метода и  поля. Позволяет встроить логику проверки данных через get и set.


 **пример свойства (работает как метод который позволяет получить значение  из поля и присвоить туда данные):**

    class  имя_класса
    {
      private int num; //поле 

      public int Num  //свойство
      {
         get{  return num; }
         set { num = value; }
      }
    }

 **пример автореализующего свойства ратотает как метод и поле в одно тоесть он является и полем и методом:**

    class  имя_класса
    {
      public int Num {get; set;} //автореаизующее свойство
    }
    
### Экземпляр класса
Реальный объект, созданный в памяти через new имеет в себе свой набор значений.

 **пример:**
 
    имя_класса название переменой = new имя_класса();

 ### Конструктор
 это метод класса который вызывается при создании экземпляра класса

 **пример:**

    class  имя_класса
    {
      public  имя_класса() //конструктор класса
      {

      }
    }


 ## 2.3 Статический класс

## 3.1 Исключение 
Исключения это  некая ошибка которая может  произойти во время   исполнения кода  например:

 * указали не правильный путь к файлу

 ### Способы обработки исключений  (блоки try/catch/finally)
   мы можем обрабатывать исключение  чтобы  наша программа не ломалась (не завершалась эестрено) с помощью ключевого слов try catch  и finally

   * **try** - это ключевое  которым мы выделяем блок кода который может кинуть нам исключений
   * **catch** - это ключевое  с помощью которого мы ловим определеный  исключения  (которые помещеные в круглые скобки)
   * **finally** - это ключевое  которое выполнит  блок кода в любом случае  даже если  кинет исключение  (или еще  вызывается если выходиим из [метода](#15-методы))
   >  может быть один блок   try  и несколько  блоков   catch которые будут ловить разные исключения   также мы можем написать  finally или же написать толко try  и finally

   **синтаксис**

      try
      {
          // тут блок кода который может кинуть исключение
      }
      catch(какое исключение ловим)
      {
          // если   мы поймали исключение попадаем  в это блок
      }
      finally
      {
          // также  если исключение произошло этот блок выполнится в любом случае если даже мы его поймали 
      }

  ### Как кидать исключения
  мы можем также кидать исключения  например если мы пишем  логику и если человек не выполняет условие а это условие критическое для дальнейшей работы мы можем кинуть исключение что чтото идет не так  это происходит с спомощью ключевого слова trow

## 3.7 Язык запросов LINQ
Linq - это язык запросов встроеный в C#  работает со всеми обьектаи которые реализуют [интрефейс IEnumerable](#36-базовые-интерфейсы) работает Linq как запрос к таблице в SQL (только с Linq это запрос к [коллекции](#34-коллекции) , [масиву](#14-массивы) и тд)  тоесть мы можем вытягивать нужные нам данные  которые будут отфильтрованы отсортированы и много чего еще

### Способы написания LINQ запроса (на примере фильтрации (WHERE))

* C помощью [методов расширения (Extension)](#метод-раширения) 
тоесть  обращение к  данным в  [коллекциях](#34-коллекции) или [масивах](#14-массивы) с помощью  методов [методов расширения (Extension)](#метод-раширения)  куда мы передаем предикат (это метод который возращает bool значение) в виде лямды  говоря как мы хотим работать с данными(не для всех методов) 

**пример на масиве**

        int[] number = {123,5,47,2,9,5,7,38,2,6}; //сам масив типа  int 

        int[] result = number.Where(x => x > 23).ToArray(); // производим фильтрацию с  помощью  метода расширения  Where  передаем лямду которая сверяет элемент на то что он больше 23 если вернет true  то  этот элемент попадет в новый масив  метод ToArray перевод эти данные в масив  чтоб мы могли их записать



* С помощью синтаксиса запроса
тоесть это происходит методом написания  как базового SQL запроса   

**пример на масиве**

        int[] number = {123,5,47,2,9,5,7,38,2,6}; //сам масив типа  int 

        int[] result = (from num in number  //берем элементы из масива
                      where num > 23 //проверяем на условие
                       select num ).ToArray() //если значение прошло условие  то  возращаем значение и записываем в  масив с помощью методв ToArray



## 3.7 Работа с данными и файлами (IO)

IO (Input/Output) операции ввода-вывода — это процессы передачи данных между оперативной памятью компьютера  и внешним миром или другими устройствами  если попростому это когда наша программа обращается к системам которые находятся в не оператички тоесть жеский диск(файлы) интернет модуль(доступ в интернет)
 
### StreamWriter 
[класс](#22-класс) который отвечает за работу над файлом  а иммено для  записис в него

 метод WriteLineAsync() — [асихронно](#35-асихронность) записывает строку и автоматически переносит курсор на новую.

 **пример**


            //делаем такую консрукцию  using  чтобы очистить память
            using(StreamWriter file = new StreamWriter("путь к файлу"))
            {
                //пишем в фаил асихронно и дожидаемся
              await file.WriteLineAsync("текст что хотим записать");
            }
        
### StreamReader 
[класс](#22-класс) который отвечает за работу над файлом  а иммено для  чтения с него 

метод ReadToEndAsync() — [асихронно](#35-асихронность) читает весь файл целиком от начала до конца в одну большую строку.
 
 **пример**

             //делаем такую консрукцию  using  чтобы очистить память
            using(StreamReader file = new StreamReader("путь к файлу")
            {
                //читаем весь фаил асихронно и дожидаемся

                return await file.ReadToEndAsync(); // воозращаем считаные данные
            }
